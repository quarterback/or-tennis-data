[`../SKILLS-COMPENDIUM.md`](../SKILLS-COMPENDIUM.md). This directory is
the same content split for working use.

## How the files are structured

Each `skills/NN-*.md` file follows the same shape:

- **One-line** summary of the skill.
- **Evidence** with file paths and line numbers.
- **Resume bullets specific to this skill.**
- **Where to grow** — concrete next moves.

Read them in any order. They're linkable from cover letters, portfolio
pages, or self-evals.

---

## How to use this

### To update your resume

Open [`RESUME-BULLETS.md`](RESUME-BULLETS.md). It's organized by likely
target role (data engineering, applied/sports analytics, software
engineering, product/staff). Pick the bullets that fit, trim or
combine, and drop them in.

The "project-summary lines" at the bottom give you the one-paragraph
header for the project entry.

### To write a portfolio page

Use the per-skill files as section sources. A solid portfolio page for
this project would lead with the project summary, then walk through
3–5 skills with one anchored evidence example each. Skills 1, 3, 4, 6,
and 12 are the most portfolio-ready.

### To plan what to learn next

Open [`LEARNING-PLAN.md`](LEARNING-PLAN.md). Six quarters, each with a
primary goal, a secondary goal, a reading list, and an artifact to
ship. Every quarter extends work you've already done — no greenfield
intimidation.

### To self-evaluate

Read the per-skill "Where to grow" sections. They're honest, not
aspirational — every gap is something the codebase actually has.

---

## A note on framing

The skills here aren't "things you've heard of." They're things you
*shipped* — usually solo, usually under self-imposed quality bars,
usually documented well enough that someone else could reproduce the
reasoning.

This is a hobby project. That makes the discipline more impressive,
not less. Most professional codebases don't have this caliber of AAR,
non-goal articulation, or root-cause data-quality work.
